# ブラウザ拡張機能用: メール認証無効化手順

ブラウザ拡張機能では確認メールのリンクをクリックできないため、Supabaseでメール認証を無効にする必要があります。

## 手順

### 1. Supabaseダッシュボードでの設定

1. Supabaseプロジェクトのダッシュボードにログイン
2. 左サイドバーから「Authentication」をクリック
3. 「Settings」タブをクリック
4. 「Email Confirmation」セクションを見つける
5. 「Enable email confirmations」のトグルを**オフ**にする
6. 「Save」ボタンをクリック

### 2. 追加のSQL設定（オプション）

より確実にするため、以下のSQLをSupabaseのSQL Editorで実行することもできます：

```sql
-- メール認証を無効化（既存ユーザーも含む）
UPDATE auth.users 
SET email_confirmed_at = NOW() 
WHERE email_confirmed_at IS NULL;
```

### 3. 確認方法

設定が正しく適用されているか確認：

1. 新しいテストアカウントを作成
2. 確認メールが送信されないことを確認
3. 登録後すぐにログイン状態になることを確認

## 注意事項

- この設定により、無効なメールアドレスでもアカウント作成が可能になります
- セキュリティを考慮し、有効なメールアドレスの使用を推奨します
- 本番環境では適切なバリデーションを実装することを検討してください

## トラブルシューティング

### 既に確認メールを受信している場合

1. 既存のアカウントを削除して新しく作成する
2. または、上記のSQLを実行して既存ユーザーの認証状態を更新する

### 設定が反映されない場合

1. ブラウザのキャッシュをクリア
2. 拡張機能を再読み込み
3. Supabaseプロジェクトの設定を再確認
